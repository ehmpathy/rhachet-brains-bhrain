# 3.3.repair.cicd.defects

## defects

### defect 1: xai integration tests fail with incorrect API key

**what**: xai integration tests failed with `400 "Incorrect API key provided: sk***gA. You can obtain an API key from https://console.x.ai."`

**why**: the test workflow was not passing the XAI_API_KEY secret to integration test jobs. the workflow had secrets for anthropic, openai, tavily, and chutes, but not xai. the rhachet-brains-xai package reads XAI_API_KEY directly from process.env (not via passed credentials), so the secret must be available as an environment variable.

**fix**:
1. added `xai-api-key` secret input to `.github/workflows/.test.yml` workflow_call declaration
2. added `XAI_API_KEY: ${{ secrets.xai-api-key }}` to env vars in both integration test steps
3. added `xai-api-key: ${{ secrets.XAI_API_KEY }}` passthrough in `.github/workflows/test.yml`

commits:
- `fix(ci): pass api keys to integration tests and prune qwen` (prior)
- `cont(ci): add xai-api-key to workflow secrets`

### defect 2: report slow tests step fails on null runtime values

**what**: the "report slow tests" step in CI failed with exit code 1 even when tests passed.

**why**: the bash procedure used `set -eu` and performed arithmetic on `.perfStats.runtime` which could be null for some test results. the expression `$((null / 1000))` or `$((' / 1000))` causes bash to fail under `set -e`.

**fix**:
1. added jq null coalesce (`// 0`) for `.perfStats.runtime` to default null to 0
2. added bash guard `[[ -z "$runtime" || "$runtime" == "null" ]] && continue` to skip invalid values

applied to both integration and acceptance test shard jobs.

commit: `cont(ci): fix report slow tests step for null runtime values`
