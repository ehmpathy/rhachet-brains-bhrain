## hazard 1: jest ESM transformation removed

**category**: test hazard

**what changed**:
- `jest.unit.config.ts`: removed `@anthropic-ai` from transformIgnorePatterns
- `jest.integration.config.ts`: removed `@anthropic-ai` and `@openai/codex-sdk` from transformIgnorePatterns
- removed `extensionsToTreatAsEsm: ['.ts']`
- removed `mjs` from moduleFileExtensions

**why it may break**:
if `@anthropic-ai/sdk` or `@openai/codex-sdk` still ship ESM-only code, jest will fail to transform them.

**verify**:
```sh
npm run test:unit
npm run test:integration
```

## hazard 2: tsconfig module lookup changed

**category**: config hazard

**what changed**:
```diff
-"module": "commonjs",
-"moduleResolution": "node",
+"module": "node16",
+"moduleResolution": "node16",
```
