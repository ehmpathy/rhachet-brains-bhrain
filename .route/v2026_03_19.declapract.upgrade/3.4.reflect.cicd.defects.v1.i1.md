## defect classifications

### defect 1: xai integration tests fail with incorrect API key

**classification**: repo quirk

**reason**: this repo uses `rhachet-brains-xai` which reads `XAI_API_KEY` from `process.env`. the workflow needed the secret added to the workflow_call chain. this is specific to this repo dependencies — not a defect in the shared practice.

### defect 2: report slow tests step fails on null runtime values

**classification**: practice bug

**reason**: the practice in declapract-typescript-ehmpathy does not handle null `.perfStats.runtime` values. the jq expression outputs raw null which causes bash arithmetic to fail under `set -eu`.

**citation**:
- upstream file: `ehmpathy/declapract-typescript-ehmpathy:src/practices/cicd-common/best-practice/.github/workflows/.test.yml`
- lines 238-249 (integration) and 307-318 (acceptance)
- search: `gh search code "report slow tests" --repo ehmpathy/declapract-typescript-ehmpathy`

**fix applied in this repo**:
- jq null coalesce: `(.perfStats.runtime // 0)` 
- bash guard: `[[ -z "$runtime" || "$runtime" == "null" ]] && continue`

**recommended action**: pr to fix null handling in upstream practice
