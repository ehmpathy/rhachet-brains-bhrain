# blocker.1: Failhide via bare catch hiding original error in test env

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.test/pitofsuccess.errors/rule.forbid.failhide.md

**locations**:
- jest.integration.env.ts:55

The try/catch around execSync catches the database connection failure but discards the original error entirely (bare catch {}) and throws a plain Error with a generic message. This is a failhide hazard: real errors are silently hidden, preventing diagnosis. Per rules, catch must only handle allowlisted errors and rethrow the rest; use MalfunctionError with cause or metadata for the original error.

**snippet**:
```ts

    throw new Error(
      `did you forget to \`npm run start:testdb\`? cant connect to database`,
    );
  }
```
