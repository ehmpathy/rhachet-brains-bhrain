### intentional patterns

**arch-hazards-maintenance blocker.3: catch pattern in tool execute**

the try/catch that returns error output instead of throwing is **intentional for brain tools**. when a brain (llm) executes a tool like "run tests" and the tests fail, we want to return the error output to the brain so it can adjust its approach. throwing would crash the brain's iteration loop.

this is the correct pattern for codewrite tool execution — the brain needs to see the error output so it can fix the code.
