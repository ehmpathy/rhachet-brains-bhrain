## blockers

| blocker | handoff | justified? |
|---------|---------|------------|
| openai tool call with optional props | `3.3.blueprint.v1.handoff.openai-schema.md` | ✅ upstream dependency issue |

### blocker justification

the openai schema issue is in `rhachet-brains-openai`, not in bhrain. OpenAI's strict mode function call API requires all properties to be in the `required` array. when zod schemas have optional properties, the conversion doesn't satisfy this requirement.

this is a "foreman possesses the key" situation - the fix must be made in the upstream package `rhachet-brains-openai`, which is outside this repo's scope.

**tried**:
1. examined the error message: `Invalid schema for function 'read': Absent 'offset'`
2. confirmed the schema has optional props: `offset: z.number().optional()`
3. verified this is OpenAI API strictness, not bhrain logic
4. confirmed anthropic works fine with same tools

**why impossible to fix here**:
- the schema conversion happens inside `rhachet-brains-openai`
- bhrain correctly passes the tool definitions
- the upstream package must handle the conversion
