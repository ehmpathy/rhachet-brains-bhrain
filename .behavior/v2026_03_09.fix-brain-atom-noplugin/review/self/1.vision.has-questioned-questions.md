### question: what extension does BrainAtom need?

**answered via comparison:**

| required | BrainAtom has | gap |
|----------|---------------|-----|
| `messages[]` with roles | `prompt` string | YES - need message array |
| `tools[]` definitions | none | YES - need tool definitions |
| `toolCalls[]` in response | structured output via schema | YES - need raw tool_use blocks |
| `maxTokens` | not exposed | minor - can default |

**conclusion: BrainAtom needs extension for tool-based agentic loops.**

the extension should add a method like:
```ts
atom.generate(input: {
  messages: Message[];
  tools: Tool[];
  maxTokens?: number;
}) => Promise<{
  message: Message;  // with toolCalls when assistant calls tools
  usage: TokenUsage;
}>
```

bhrain cannot work around this via ask(). tools must be passed to the LLM natively, not serialized into prompts.
