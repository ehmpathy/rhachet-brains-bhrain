**extracting tool calls (lines 105-113):**

- ✅ handles null calls
- ✅ handles empty tools array (via `?? null` on empty map)
- ✅ field maps match contract

## tool definition transformation verification

### vision contract

```ts
interface BrainPlugToolDefinition {
  slug: string;
  description: string;
  schema: { input: z.Schema<unknown> };
}
```

### implementation trace (lines 22-29)

```ts
const toRhachetToolDefinitions = (
  tools: BrainArch1ToolDefinition[],
): BrainPlugToolDefinition[] =>
  tools.map((tool) => ({
    slug: tool.name,              // ✅ maps name → slug
    description: tool.description, // ✅ passthrough
    schema: { input: z.any() },    // ✅ schema.input present
  }));
```

**note on schema:** uses `z.any()` intentionally - comment explains: "tool schema already validated at bhrain level". this avoids double validation.

**verdict:** ✅ tool definition transformation matches contract
