## why buildPromptFromMessages is bhrain-specific

bhrain's BrainArch1SessionMessage has:
- `role: 'system' | 'user' | 'assistant' | 'tool'`
- `content: string | null`
- `toolCalls: BrainArch1ToolCall[] | null`
- `toolCallId: string | null`

rhachet's atom.ask() expects:
- `prompt: string`

the conversion is bhrain-specific because it knows how to serialize bhrain's message format to a string the LLM can understand. rhachet doesn't provide this because different repls may have different message formats.
