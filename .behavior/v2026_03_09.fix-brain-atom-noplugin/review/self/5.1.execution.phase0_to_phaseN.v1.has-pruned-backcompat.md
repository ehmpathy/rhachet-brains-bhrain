### deep examination: conversion functions

reviewed `generateBrainArch1LlmResponse.ts` for hidden backcompat:

**buildPromptFromMessages()**: converts BrainArch1SessionMessage[] to string prompt
- question: is this backcompat for the old message-based API?
- answer: no, this is required because rhachet's atom.ask() expects string prompt, not messages. this is an interface bridge, not backwards compat.

**toRhachetToolDefinitions()**: converts BrainArch1ToolDefinition[] to BrainPlugToolDefinition[]
- question: is this maintaining the old tool interface?
- answer: no, bhrain still uses its own internal tool types. conversion is required to call rhachet's atom.ask() with plugs.tools.

### vision guidance

from `1.vision.md`:
> "no custom atom interface. bhrain registers repl, rhachet handles slug."
> "zero custom atoms: bhrain uses rhachet's BrainAtom directly"

the vision explicitly requested removal of custom atoms. no backwards compat was requested.

### version context

this is version 0.1.0 (pre-1.0). semver allows incompatible changes in 0.x versions. the vision requested a clean break.

## conclusion

✓ **passed** - no backwards compat shims added. all conversion code is for interface bridging, not backwards compat. clean break as requested.
