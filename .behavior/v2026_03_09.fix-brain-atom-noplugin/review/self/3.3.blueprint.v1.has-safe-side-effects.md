# self-review: has-safe-side-effects

## the question

for side effects in the blueprint, ask:
- are external calls safe (rate limited, retry-safe)?
- are resources cleaned up (connections, files)?
- are concurrent operations safe (no race conditions)?

## side effects in blueprint

### 1. BrainAtom.ask() - external API call

**side effect:** calls LLM provider API (e.g., anthropic)

**safe?** YES.

**analysis:**
- rate limits: handled by rhachet/provider SDK
- retry-safe: yes, ask() is idempotent
- resource cleanup: handled by rhachet

**verdict:** delegated to rhachet. safe.

### 2. tool execution - external operations

**side effect:** tools may read/write files, call APIs, etc.

**safe?** DEPENDS on tool.

**analysis:**
- tool safety is tool author's responsibility
- BrainArch1 provides sandbox via toolbox permissions
- marked `[○]` retain - no change in blueprint

**verdict:** tool safety unchanged. out of scope.

### 3. episode state - in-memory accumulation

**side effect:** episode grows with each exchange

**safe?** YES.

**analysis:**
- episode is passed as input, not shared state
- no race condition: single-threaded per invocation
- memory bounded by max iterations

**verdict:** safe. no shared mutable state.

## concurrent operations

### race conditions

**searched for:**

**found:** none

**analysis:**
- BrainArch1 loop is sequential
- each invocation is isolated
- no shared state between invocations

**verdict:** no race conditions.
