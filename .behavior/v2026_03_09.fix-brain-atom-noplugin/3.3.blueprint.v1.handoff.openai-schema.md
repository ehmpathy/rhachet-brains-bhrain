# handoff: rhachet-brains-openai schema defect

## the defect

OpenAI integration tests fail with schema validation error when tools have optional properties.

```
400 Invalid schema for function 'read': In context=(), 'required' is required
to be supplied and to be an array that includes every key in properties. Absent: 'offset'.
```

## root cause

OpenAI's strict mode function call requires ALL properties to be in the `required` array. When a zod schema has optional properties like:

```ts
const schemaReadInput = z.object({
  path: z.string(),
  offset: z.number().optional(),  // ← optional
  limit: z.number().optional(),   // ← optional
});
```

The JSON schema conversion produces:
```json
{
  "properties": {
    "path": { "type": "string" },
    "offset": { "type": "number" },
    "limit": { "type": "number" }
  },
  "required": ["path"]  // ← OpenAI rejects this
}
```

OpenAI expects either:
1. All properties in `required`, OR
2. Optional properties with nullable types: `"type": ["number", "null"]`

## where it manifests

- **test failed**: `src/domain.operations/arch1/core/invokeBrainArch1.integration.test.ts`
- **case**: `[case4] openai provider with file read tool`
- **tool with optional props**: `src/domain.operations/arch1/plugins/toolboxes/files/read.ts`

## ownership

**this is NOT a bhrain defect** — bhrain correctly uses rhachet's contracts.

**this IS a rhachet-brains-openai defect** — the atom's schema conversion doesn't handle optional properties for OpenAI's strict mode.

## fix location

`rhachet-brains-openai` needs to transform optional zod properties to OpenAI-compatible format:

```ts
// before (current broken behavior)
{ "type": "number" }  // in properties
// not in required array

// after (openai-compatible)
{ "type": ["number", "null"] }  // in properties
// added to required array
```

## interim action

skip the OpenAI test in bhrain until rhachet-brains-openai is fixed:

```ts
given.skip('[case4] openai provider with file read tool', () => {

  // openai strict mode requires all properties in required array
```
