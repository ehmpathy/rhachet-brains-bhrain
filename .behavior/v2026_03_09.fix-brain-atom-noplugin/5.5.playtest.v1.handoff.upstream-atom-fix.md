# handoff: upstream atom fix for tool use + structured output

## summary

open-source models on together ai and chutes **call tools correctly**, but return plain text final responses that fail JSON parse.

**fix**: upstream atoms need a plain text fallback in their JSON parse block.

## the problem

when tools are present, the upstream atoms conditionally omit `response_format`. after tool execution completes, models return plain text like "Done!" instead of JSON.

```
model call with tools → returns tool_calls ✓
tool execution → success ✓
model final response → "Done! I wrote the file." (plain text)
bhrain JSON.parse() → SyntaxError ✗
```

## the fix

in the atom's response handler:

```js
let parsed;
try {
  parsed = JSON.parse(content);
} catch {
  // model returned plain text - use as-is for string schemas
  parsed = content;
}
const output = askInput.schema.output.parse(parsed);
```

this works because bhrain uses `z.string().nullable()` schema - any text is valid.

## why not always send response_format?

open-source models (DeepSeek, Kimi) prioritize `json_schema` over tool calls. when both `response_format` and `tools` are sent, they refuse to call tools and return JSON directly.

xai/grok handles both `response_format` + `tools` together. open-source models cannot.
