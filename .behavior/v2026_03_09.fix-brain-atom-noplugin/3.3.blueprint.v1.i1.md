# blueprint: bhrain conforms to rhachet contracts

## overview

bhrain uses rhachet's contracts directly:
- **BrainAtom**: no custom atom plugins
- **BrainPlugToolDefinition**: no custom tool definitions

rhachet's `genContextBrain({ choice: { repl: 'bhrain/arch1@claude/sonnet' } })` discovers bhrain's repl + the atom, composes them together.

## status: ready to implement

rhachet has released tool support and `genBrainPlugToolDeclaration`. bhrain can now implement against the actual contracts.

## rhachet contract (released)

```ts
// BrainAtom.ask() signature
ask<TOutput, TPlugs extends BrainPlugs>(input: {
  on?: { episode: BrainEpisode };
  plugs?: TPlugs;
  role: { briefs?: Artifact[] };
  prompt: string;
  schema: { output: z.Schema<TOutput> };
}) => Promise<BrainOutput<TOutput, 'atom', TPlugs>>

// BrainPlugs - tools go in plugs
interface BrainPlugs {
  tools?: BrainPlugToolDefinition[];
}

// BrainOutput - calls come out (typed based on TPlugs)
interface BrainOutput<TOutput, TGrain, TPlugs> {
  output: TOutput | null;  // null when brain defers to tool calls
  calls: { tools: BrainPlugToolInvocation[] } | null;
  metrics: BrainOutputMetrics;
  episode: BrainEpisode;
}

// tool definition
interface BrainPlugToolDefinition {
  slug: string;
  description: string;
  schema: { input: z.Schema<unknown> };
}

// tool invocation
interface BrainPlugToolInvocation {
  exid: string;   // provider-assigned ID for correlation
  slug: string;
  input: unknown;
}
```

**why plugs.tools?** tools are plugins - you plug tools into the brain to use them.

episode continuation (`on.episode`) handles conversation history. tools enable native LLM tool call support.

2. **calls.tools** - `calls: { tools: BrainPlugToolInvocation[] } | null` in BrainOutput вњ“

## key implication

when BrainAtom supports tool use, **any BrainAtom that implements the rhachet-brains contract becomes usable with this BrainRepl**.

one interface. many providers. zero custom plugins.
