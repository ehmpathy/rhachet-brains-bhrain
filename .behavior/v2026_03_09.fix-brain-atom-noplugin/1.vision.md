# vision: bhrain conforms to rhachet's BrainAtom contract

## the core insight

**rhachet's BrainAtom IS the generalization of BrainArch1's atom plugins.**

the composite slug `bhrain/arch1@claude/sonnet` works with rhachet's discovery:

```ts
genContextBrain({ choice: { repl: 'bhrain/arch1@claude/sonnet' } })
```

rhachet:
- parses the composite slug (splits on @)
- discovers bhrain's repl
- discovers the atom
- composes them together

bhrain just registers itself as a repl provider. no manual imports. no custom atom interface.

see also: `.agent/repo=.this/role=any/briefs/define.bhrain-composable-repl.md`

## the outcome world

### before
```
rhachet-brains-bhrain/
```

## user experience

### before: manual composition

```ts
import { genAtomAnthropic } from 'rhachet-brains-bhrain/plugins/atoms/anthropic';
import { genBrainRepl } from 'rhachet-brains-bhrain';

const atom = genAtomAnthropic({ model: 'claude-sonnet-4-20250514' });
const brain = await genBrainRepl({ slug: 'bhrain/arch1@claude/sonnet', atom, context });
```

### after: one call

```ts
import { genContextBrain } from 'rhachet';

const ctx = await genContextBrain({ choice: { repl: 'bhrain/arch1@claude/sonnet' } });
// done - rhachet discovered bhrain repl + claude atom, composed them
```

## evaluation

### pros
- **zero custom atoms**: bhrain uses rhachet's BrainAtom directly
- **one call**: genContextBrain handles all discovery and composition
- **composite slug**: repl + atom choice in one string
- **automatic upgrades**: new atoms work without bhrain changes

### cons
- **refactor required**: BrainArch1 internals must use BrainAtom.ask() instead of generate()

### the key principle

**rhachet is the source of truth. bhrain conforms.**

## rhachet tool support (released)

rhachet now supports tool use natively. the contract:

```ts
// BrainAtom.ask() signature
ask<TOutput, TPlugs extends BrainPlugs>(input: {
  on?: { episode: BrainEpisode };
  plugs?: TPlugs;
  role: { briefs?: Artifact[] };
  prompt: string;
  schema: { output: z.Schema<TOutput> };
}) => Promise<BrainOutput<TOutput, 'atom', TPlugs>>

// BrainPlugs - tools go in plugs
interface BrainPlugs {
  tools?: BrainPlugToolDefinition[];
}

// BrainOutput - calls come out
interface BrainOutput<TOutput, TGrain, TPlugs> {
  output: TOutput | null;  // null when brain defers to tool calls
  calls: { tools: BrainPlugToolInvocation[] } | null;
  metrics: BrainOutputMetrics;
  episode: BrainEpisode;
}

// tool definition
interface BrainPlugToolDefinition {
  slug: string;
  description: string;
  schema: { input: z.Schema<unknown> };
}

// tool invocation
interface BrainPlugToolInvocation {
  exid: string;   // provider-assigned ID for correlation
  slug: string;
  input: unknown;
}
```

**key implication**: any BrainAtom that implements the rhachet-brains contract works with this BrainRepl. one interface, many providers, zero custom plugins.

## rhachet tool declarations (released)

rhachet also provides tool declaration utilities. bhrain should use them instead of custom tool definitions.

```ts
// rhachet provides genBrainPlugToolDeclaration factory
import { genBrainPlugToolDeclaration } from 'rhachet/brains';

const readTool = genBrainPlugToolDeclaration({
  slug: 'read',
  name: 'read',
  description: 'Reads file contents at specified path',
  schema: {
    input: z.object({ path: z.string(), offset: z.number().optional() }),
    output: z.object({ content: z.string() }),
  },
  execute: async ({ invocation }) => {
    const content = await fs.readFile(invocation.input.path, 'utf-8');
    return { content };
  },
});
```

**before**: bhrain defined `BrainArch1ToolDefinition` + separate execute functions + `toJsonSchema` helper
**after**: use `genBrainPlugToolDeclaration` - definition and execution unified, zod schemas passed directly

### the key principle (extended)

**rhachet is the source of truth. bhrain conforms.**
- atoms: use rhachet's BrainAtom (not custom atom plugins)
- tools: use rhachet's BrainPlugToolDefinition (not custom tool definitions)
