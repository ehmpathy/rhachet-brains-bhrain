# blackbox criteria: eject BrainArch1 into rhachet-brains-bhrain

> experience boundaries for the ejection

---

# usecase.1 = install and import

```
given a developer wants to use bhrain's agentic brain
  when they install the package
    then `npm install rhachet-brains-bhrain` succeeds
      sothat the package is available in node_modules
  when they import from the package
    then `import { genBrainRepl } from 'rhachet-brains-bhrain'` succeeds
      sothat they can access the brain api
    then typescript types are available
      sothat they get autocomplete and type checks
```

---

# usecase.2 = configure and invoke brain

```
given a developer has imported genContextBrain
  when they select a brain via composite slug
    then genContextBrain({ choice: 'bhrain/arch1@claude/sonnet' }) returns BrainRepl
      sothat they get a repl with atom baked in
    then genContextBrain({ choice: 'bhrain/arch1@xai/grok' }) returns BrainRepl
      sothat they can use any installed rhachet-brains-* atom
  when they invoke brain.ask() for readonly analysis
    then brain.ask({ role, prompt, schema }) returns BrainOutput
    then output.output conforms to the provided schema
    then output.metrics contains token counts and cost
    then output.episode contains continuation handle
      sothat they can analyze without side effects
  when they invoke brain.act() for read+write actions
    then brain.act({ role, prompt, schema }) returns BrainOutput
    then the brain can execute write tools (Edit, Write, Bash)
      sothat they can make changes to the codebase
```

---

# usecase.3 = conversation continuation

```
given a developer has invoked the brain once
  when they want to continue the conversation
    then they pass on: { episode: priorResult.episode } to the next call
    then the brain has context of the prior exchange
      sothat multi-turn workflows are supported
```

---

# usecase.4 = readonly vs read+write modes

```
given a developer wants to control what the brain can do
  when they use brain.ask()
    then only readonly tools are available (Read, Glob, Grep)
    then write tools are forbidden (Edit, Write, Bash)
      sothat the brain can analyze without side effects
  when they use brain.act()
    then full tool access is available
    then the brain can read, write, and execute
      sothat the brain can make changes
```

---

# usecase.5 = provider agnostic (composite slug pattern)

```
given BrainArch1 accepts any BrainAtom via composite slug
  when the caller uses rhachet-brains-anthropic
    then they use slug 'bhrain/arch1@claude/sonnet'
  when the caller uses rhachet-brains-openai
    then they use slug 'bhrain/arch1@openai/gpt-4o'
  when the caller uses rhachet-brains-xai
    then they use slug 'bhrain/arch1@xai/grok'
  when the caller uses slug 'bhrain/arch1' alone
    then BadRequestError is thrown
      sothat repl slug without atom is invalid
      sothat callers choose their preferred provider via @$atom suffix
      sothat this package has no direct dependencies on brain suppliers
```

---

# usecase.6 = error cases

```
given a developer invokes the brain
  when input exceeds token limit
    then BadRequestError is thrown before the loop starts
      sothat the developer knows to reduce input size
  when the brain cannot complete within constraints
    then the output reflects partial progress
      sothat the developer can continue or adjust
  when an error occurs mid-loop
    then the error is surfaced with context
      sothat the developer can diagnose and fix
```

---

# usecase.7 = package identity

```
given this repo is a clone of rhachet-brains-anthropic
  when the ejection is complete
    then package.json name = 'rhachet-brains-bhrain'
    then package.json description references bhrain
    then the sdk exports genBrainRepl (not invokeBrainArch1)
      sothat the package identity is bhrain, not anthropic
```

# usecase.8 = build and test

```
given a developer clones the repo after ejection
  when they run npm install
    then all dependencies install successfully
  when they run npm run build
    then the build succeeds with no errors
  when they run npm run test
    then all tests pass
      sothat the ejection is verified as complete and functional
```
