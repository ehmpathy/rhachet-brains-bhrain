## deviations found

### deviation 1: vision shows BrainArch1Atom but blueprint exports genAtom*

**vision example shows:**
```ts
import { invokeBrainArch1, BrainArch1Actor, BrainArch1Atom } from 'rhachet-brains-bhrain';

const actor = new BrainArch1Actor({
  atom: new BrainArch1Atom({ provider: 'anthropic', model: '...' }),
  ...
});
```

**blueprint public api exports:**
```ts
export { genAtomAnthropic } from './domain.operations/arch1/plugins/atoms/anthropic';
export { genAtomOpenai } from './domain.operations/arch1/plugins/atoms/openai';
export { genAtomQwen } from './domain.operations/arch1/plugins/atoms/qwen';
// NO BrainArch1Atom export
```

**analysis:**
- vision shows direct `new BrainArch1Atom({ provider, model })` instantiation
- blueprint exports generator functions: `genAtomAnthropic({ model })`
- these are different patterns

**question:** which is correct?

**answer:** the `genAtom*` pattern is the actual implementation — providers are separate functions, not a single class with a provider parameter.

**action:** vision needs update to show correct pattern:
```ts
import { invokeBrainArch1, BrainArch1Actor, genAtomAnthropic } from 'rhachet-brains-bhrain';

const actor = new BrainArch1Actor({
  atom: genAtomAnthropic({ model: 'claude-3-5-sonnet-latest' }),
  ...
});
```

**status:** **FLAGGED** — vision documentation needs update (not blueprint deviation)
