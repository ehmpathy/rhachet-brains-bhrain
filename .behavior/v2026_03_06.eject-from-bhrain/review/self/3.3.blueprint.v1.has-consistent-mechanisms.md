## duplication analysis

### genBrainAtom vs genAtomAnthropic

| aspect | genBrainAtom (current) | genAtomAnthropic (BrainArch1) |
|--------|------------------------|-------------------------------|
| base class | rhachet BrainAtom | BrainArch1Atom |
| api | Anthropic Messages API | Anthropic Messages API |
| purpose | standalone atom | BrainArch1 plugin |

**is this duplication?** NO — these are different systems:
- genBrainAtom creates atoms for rhachet's brain framework
- genAtomAnthropic creates atoms for BrainArch1's brain framework

**result:** genBrainAtom is REMOVED (phase.2), genAtomAnthropic is ADDED (phase.3)

### genBrainRepl vs invokeBrainArch1

| aspect | genBrainRepl (current) | invokeBrainArch1 (BrainArch1) |
|--------|------------------------|-------------------------------|
| base class | rhachet BrainRepl | BrainArch1Actor |
| api | Claude Agent SDK | custom loop with atoms |
| purpose | claude code wrapper | provider-agnostic brain loop |

**is this duplication?** NO — different architectures:
- genBrainRepl wraps Claude Agent SDK
- invokeBrainArch1 runs custom agentic loop with pluggable atoms

**result:** genBrainRepl is REMOVED (phase.2), invokeBrainArch1 is ADDED (phase.3)
