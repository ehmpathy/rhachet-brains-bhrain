## mechanisms in genBrainRepl.ts

### 1. parseCompositeSlug (lines 64-73)

**question**: does this duplicate an extant parser?

**search conducted**:
- searched rhachet package for slug parsers: none found for composite slugs
- searched this codebase: `grep -r "parseSlug\|splitSlug" src/` — no matches
- examined rhachet-brains-anthropic: uses simple slugs like `claude/sonnet`

**answer**: no. composite slugs (`$repl@$atom`) are unique to bhrain. rhachet does not parse composite slugs because:
1. other brain packages (anthropic, openai) use simple slugs
2. composite slug is a bhrain-specific pattern to bake atom choice into repl

**implementation detail**: the parser is minimal (10 lines), handles only the `@` separator:
```ts
const atIndex = input.slug.indexOf('@');
if (atIndex === -1) return { replSlug: input.slug, atomSlug: null };
return {
  replSlug: input.slug.slice(0, atIndex),
  atomSlug: input.slug.slice(atIndex + 1),
};
```

**verdict**: ✅ unique mechanism, no duplication

### 2. buildZeroMetrics

**question**: should we call `calcBrainOutputCost` from rhachet instead?

**analysis**:
- `calcBrainOutputCost` computes actual cost from spec + token counts
- `buildZeroMetrics` returns placeholder zeros
- these serve different purposes

**why zeros**: BrainArch1 internally tracks `totalTokenUsage` but does not track cost. proper cost calculation requires:
1. knowing which atom was invoked (for its spec)
2. summing across all iterations

this is future work. for now, token counts are accurate; cost is placeholder.

**verdict**: ✅ deliberate placeholder — not a missed reuse opportunity

**note**: when proper cost calc is added, use `calcBrainOutputCost` from rhachet
