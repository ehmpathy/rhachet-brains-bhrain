#### rule.require.get-set-gen-verbs

**check:** operations use get, set, or gen prefix

**blueprint exports:**
- invokeBrainArch1 — uses invoke* prefix (exempt for entry points)
- genAtomAnthropic — ✓ uses gen*
- genAtomOpenai — ✓ uses gen*
- genAtomQwen — ✓ uses gen*
- **toolboxFiles** — uses toolbox* prefix ⚠️
- **toolboxBash** — uses toolbox* prefix ⚠️
- **toolboxWeb** — uses toolbox* prefix ⚠️

**analysis for invoke*:**
- rule says: "exempt: contract/cli entry points (e.g., invokeAct)"
- invokeBrainArch1 is an entry point operation
- invoke* is appropriate for imperative commands
- **status:** ✓ ADHERES

**analysis for toolbox*:**
- exported from: `./domain.operations/arch1/plugins/toolboxes/*`
- rule says: "applies to: all operations in domain.operations/"
- toolbox* is NOT get/set/gen

**question:** are toolboxFiles, toolboxBash, toolboxWeb operations or objects?

- if **operations** (functions): should be `genToolboxFiles`, `getToolboxFiles`, etc.
- if **objects** (configs): rule doesn't apply (objects are not operations)

**status:** **FLAGGED** — needs verification of whether toolbox* are functions or objects
