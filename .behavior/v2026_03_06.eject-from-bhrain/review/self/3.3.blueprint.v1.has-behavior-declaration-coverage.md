## gaps found

### gap 1: BrainArch1Atom not exported

**vision shows:**
```ts
import { invokeBrainArch1, BrainArch1Actor, BrainArch1Atom } from 'rhachet-brains-bhrain';
```

**blueprint public api:**
```ts
// no BrainArch1Atom export
export { genAtomAnthropic } from './domain.operations/arch1/plugins/atoms/anthropic';
export { genAtomOpenai } from './domain.operations/arch1/plugins/atoms/openai';
export { genAtomQwen } from './domain.operations/arch1/plugins/atoms/qwen';
```

**analysis:** vision shows direct BrainArch1Atom instantiation, but blueprint exports generators (genAtomAnthropic, etc.) not the class.

**question:** which pattern is correct?
- vision pattern: `new BrainArch1Atom({ provider: 'anthropic', model: '...' })`
- blueprint pattern: `genAtomAnthropic({ model: '...' })`

**action:** check actual BrainArch1 implementation to determine correct pattern
