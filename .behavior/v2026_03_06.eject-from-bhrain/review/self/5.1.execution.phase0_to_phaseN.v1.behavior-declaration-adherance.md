## deviations found

### 1. input signature

**blueprint suggested**:
```ts
genBrainRepl({ slug: 'bhrain/arch1@claude/sonnet' })
```

**actual implementation**:
```ts
genBrainRepl({
  slug: 'bhrain/arch1@claude/sonnet',
  atom: BrainArch1Atom,
  context: BrainArch1Context,
})
```

**why**: blueprint assumed atom could be discovered from slug alone. implementation requires caller to provide atom and context because:
1. BrainArch1 needs `generate()` interface, not rhachet's `ask()`
2. atom discovery via rhachet would return wrong interface
3. context is required for tool execution (workDir, log)

**verdict**: intentional deviation — documented in has-pruned-backcompat

### 2. metrics placeholder

**vision says**:
> output.metrics contains token counts and cost

**actual**:
- token counts: accurate
- cost: placeholder zeros

**why**: proper cost calculation requires atom spec which varies by provider. placeholder is acceptable for v1.

**verdict**: partial implementation — documented in has-consistent-mechanisms
