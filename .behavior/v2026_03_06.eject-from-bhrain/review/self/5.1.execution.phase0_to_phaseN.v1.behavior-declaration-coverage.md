## usecase.3 = conversation continuation

| requirement | covered? | evidence |
|-------------|----------|----------|
| pass on: { episode } to continue | ⚠️ | episode returned but not accepted as input yet |
| brain has context of prior exchange | ⚠️ | not implemented in v1 |

**analysis**: the criteria specifies continuation via `on: { episode }`. current implementation:
- returns episode with hash and exid
- does NOT accept episode as input to continue

**why**: BrainArch1 has internal conversationHistory but translation of BrainEpisode to conversationHistory format requires additional work.

**verdict**: ⚠️ partial coverage — episode returned, continuation not wired

**open question**: should continuation be added before stone can be passed, or is this acceptable for v1?

**note**: the vision explicitly states this is a v1 ejection. episode data is preserved for future continuation support.
